# 🤖 AI 챗봇 빌더 스킬

이 스킬은 BSD 바이브코딩 수강생들이 **비즈니스용 AI 챗봇**을 쉽고 빠르게 만들 수 있도록 돕습니다.

## 📋 이 스킬이 하는 일

1. **목적 정의**: 챗봇의 역할과 목표 설정
2. **시나리오 설계**: 대화 흐름도 작성
3. **지식베이스 구축**: FAQ, 제품 정보 등 학습 데이터 준비
4. **플랫폼 선택**: 최적의 챗봇 빌더 추천
5. **배포 가이드**: 웹사이트, 메신저 연동 방법

## 🎯 언제 이 스킬을 사용하나요?

- "24/7 고객 문의 응대 챗봇을 만들고 싶어요"
- "웹사이트 방문자에게 자동으로 제품 추천해주는 봇이 필요해요"
- "FAQ 자동 응답 시스템을 구축하고 싶어요"
- "리드 정보를 수집하는 대화형 폼을 만들고 싶어요"

## 🛠️ 추천 챗봇 플랫폼

### Voiceflow ⭐⭐⭐ (최고 추천)
**장점**:
- 시각적 캔버스로 직관적 설계
- GPT-4 통합 지원
- 음성 + 텍스트 챗봇 모두 가능
- 풍부한 템플릿
- Zapier, Slack 등 다양한 연동

**단점**:
- 무료 플랜 제한적 (프로토타입만)
- 유료 플랜 $40/월~

**추천 대상**:
- 복잡한 대화 플로우 필요
- 음성 인터페이스 원함
- 프로페셔널한 챗봇 필요

### Chatbase ⭐⭐ (간편 추천)
**장점**:
- 웹사이트 URL만으로 자동 학습
- PDF, DOC 파일 업로드 가능
- 5분 안에 배포 가능
- 합리적인 가격 ($19/월~)

**단점**:
- 대화 플로우 커스터마이징 제한적
- GPT 기반이라 답변 제어 어려움

**추천 대상**:
- 빠른 프로토타입 필요
- 문서 기반 Q&A 챗봇
- 기술 지식 최소화 원함

### Botpress ⭐⭐⭐ (개발자 추천)
**장점**:
- 오픈소스 (무료)
- 자체 호스팅 가능
- 완전한 커스터마이징
- NLU 엔진 내장

**단점**:
- 기술적 지식 필요
- 설정 시간 오래 걸림

**추천 대상**:
- 개발 역량 있는 사용자
- 데이터 프라이버시 중요
- 무제한 확장 필요

### Landbot ⭐⭐ (랜딩페이지 특화)
**장점**:
- 대화형 랜딩페이지 특화
- 아름다운 UI/UX
- 리드 수집 최적화
- Stripe 결제 연동

**단점**:
- 가격대 높음 ($30/월~)
- 복잡한 로직 제한적

**추천 대상**:
- 리드 생성 목적
- 시각적으로 멋진 챗봇 원함
- 결제까지 통합하고 싶음

## 💡 챗봇 유형별 활용 사례

### 1. 고객 지원 봇 (Customer Support Bot)
```
목적: 반복적인 고객 문의 자동 응대

주요 기능:
- FAQ 자동 답변
- 주문 상태 조회
- 환불/교환 안내
- 실시간 상담원 연결

효과:
- 응대 시간 70% 감소
- 24/7 서비스 제공
- 상담원 업무 부담 감소
- 고객 만족도 향상
```

### 2. 영업 봇 (Sales Bot)
```
목적: 리드 발굴 및 제품 추천

주요 기능:
- 방문자 니즈 파악 질문
- 제품/서비스 맞춤 추천
- 가격 견적 자동 산출
- 데모/상담 예약
- CRM 자동 연동

효과:
- 리드 전환율 40% 증가
- 영업 파이프라인 자동화
- 24시간 리드 수집
```

### 3. 온보딩 봇 (Onboarding Bot)
```
목적: 신규 사용자 가이드

주요 기능:
- 단계별 튜토리얼 제공
- 자주 묻는 질문 선제 답변
- 개인화된 설정 가이드
- 다음 액션 제안

효과:
- 활성 사용자 60% 증가
- 지원 티켓 50% 감소
- 사용자 경험 향상
```

### 4. 예약 봇 (Booking Bot)
```
목적: 예약/스케줄링 자동화

주요 기능:
- 가능한 날짜/시간 확인
- 실시간 예약 처리
- 캘린더 자동 연동
- 리마인더 발송

효과:
- 예약 처리 시간 90% 감소
- 노쇼율 감소
- 운영 효율성 증가
```

### 5. 교육 봇 (Education Bot)
```
목적: 학습 지원 및 평가

주요 기능:
- 대화형 학습 콘텐츠
- 퀴즈 및 평가
- 개인화된 학습 경로
- 진도 트래킹

효과:
- 학습 참여도 80% 증가
- 개별 맞춤 교육 가능
- 학습 데이터 자동 수집
```

## 📐 챗봇 메시지 작성 가이드

### 1. 환영 메시지
```
❌ 나쁜 예:
"안녕하세요"

✅ 좋은 예:
"안녕하세요! 👋 [브랜드명] 챗봇입니다.
무엇을 도와드릴까요?

1. 제품 문의
2. 주문 조회
3. 상담 신청
4. FAQ 보기

번호를 선택하거나 자유롭게 질문해주세요!"
```

### 2. 명확한 선택지 제공
```
❌ 나쁜 예:
"어떤 제품이 궁금하신가요?"

✅ 좋은 예:
"어떤 제품이 궁금하신가요?

🔹 노트북
🔹 태블릿
🔹 스마트폰
🔹 액세서리

버튼을 눌러 선택해주세요!"
```

### 3. 단계적 정보 수집
```
❌ 나쁜 예:
"이름, 이메일, 전화번호, 관심 제품을 모두 알려주세요"

✅ 좋은 예:
Step 1: "먼저 성함을 알려주시겠어요?"
    ↓
Step 2: "감사합니다, {{name}}님! 이메일 주소를 입력해주세요."
    ↓
Step 3: "거의 다 됐어요! 연락 가능한 번호를 알려주세요."
```

### 4. 공감과 개인화
```
❌ 나쁜 예:
"주문이 지연되고 있습니다"

✅ 좋은 예:

```

### 5. 명확한 다음 단계
```
❌ 나쁜 예:
"답변이 도움이 되셨나요?"

✅ 좋은 예:
"답변이 도움이 되셨나요?

👍 네, 해결됐어요
👎 아니요, 더 도움이 필요해요
💬 상담원과 대화하고 싶어요"
```

## 🔧 Voiceflow로 챗봇 만들기

### Step 1: 프로젝트 생성
```
1. Voiceflow 가입 후 "Create Project"
2. 템플릿 선택 또는 "Start from Scratch"
3. 챗봇 이름 및 설정
```

### Step 2: 환영 블록 설정
```
1. "Start" 블록 더블 클릭
2. "Speak" 블록 추가
3. 환영 메시지 작성:
   "안녕하세요! 무엇을 도와드릴까요?"
```

### Step 3: 의도 파악 (Intent)
```
1. "Choice" 또는 "Buttons" 블록 추가
2. 옵션 설정:
   - 제품 문의
   - 주문 조회
   - 상담 신청
3. 각 선택지를 다음 블록으로 연결
```

### Step 4: 조건 로직 추가
```
1. "IF" 블록 추가
2. 조건 설정 예시:
   IF {user_type} = "신규고객"
      → 온보딩 플로우
   ELSE
      → 일반 응대 플로우
```

### Step 5: API 연동 (고급)
```
1. "API" 블록 추가
2. 엔드포인트 설정:
   - Method: GET/POST
   - URL: https://your-api.com/orders
   - Headers: API Key
3. 응답 데이터 변수로 저장
4. "Speak" 블록에서 변수 사용:
   "주문 번호 {{order_id}}는 {{status}} 상태입니다."
```

### Step 6: GPT 통합
```
1. "AI Set" 블록 추가
2. GPT 모델 선택 (GPT-4 추천)
3. 시스템 프롬프트 작성:
   "당신은 친절한 고객 지원 챗봇입니다. 
    항상 정중하고 공감하는 답변을 하세요."
4. Knowledge Base 연결 (선택사항)
```

### Step 7: 테스트 및 배포
```
1. 우측 "Test" 패널에서 대화 시뮬레이션
2. 모든 경로 테스트
3. "Publish" 클릭
4. 웹사이트 임베드 코드 복사
```

## 💻 웹사이트 임베드 코드

### 기본 HTML 임베드
```html
<!-- Voiceflow 챗봇 -->
<script type="text/javascript">
  (function(d, t) {
      var v = d.createElement(t), s = d.getElementsByTagName(t)[0];
      v.onload = function() {
        window.voiceflow.chat.load({
          verify: { projectID: 'YOUR_PROJECT_ID' },
          url: 'https://general-runtime.voiceflow.com',
          versionID: 'production'
        });
      }
      v.src = "https://cdn.voiceflow.com/widget/bundle.mjs"; 
      v.type = "text/javascript"; 
      s.parentNode.insertBefore(v, s);
  })(document, 'script');
</script>
```
